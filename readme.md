تشغيل متصفح داخل **GitHub Codespaces** فكرة ذكية جداً، لأنك بذلك تقوم بعملية "Inception" تقنية: متصفح (جهازك) يفتح متصفحاً (Codespaces) الذي بدوره يفتح متصفحاً ثالثاً (الذي سنقوم بتثبيته).
بما أن Codespaces تعتمد على حاويات لينكس وتدعم **Docker** و**Port Forwarding**، فإليك أسرع الطرق للقيام بذلك:
### الطريقة الأولى: استخدام Docker (الأسرع والأسهل)
بما أن Docker مثبت مسبقاً في معظم بيئات Codespaces، يمكنك تشغيل متصفح Firefox بواجهة ويب مباشرة.
 1. افتح الـ Terminal في Codespaces.
 2. قم بلصق الأمر التالي:
   ```bash
   docker run -d \
     --name=firefox-container \
     -p 3000:3000 \
     -e PUID=1000 \
     -e PGID=1000 \
     -e TZ=Etc/UTC \
     lscr.io/linuxserver/firefox:latest
   
   ```
 3. بمجرد تشغيل الحاوية، ستظهر لك رسالة من VS Code في الزاوية اليمنى بالأسفل تقول:
   > "A service is running on port 3000. **Open in Browser**"
   > 
 4. اضغط على **Open in Browser**. سيفتح لك لسان تبويب جديد يحتوي على متصفح Firefox كامل يعمل داخل البيئة.
### الطريقة الثانية: استخدام نظام Kasm (الأكثر استقراراً)
إذا كنت تريد تجربة أكثر سلاسة واحترافية:
 1. شغّل هذا الأمر في الـ Terminal:
   ```bash
   docker run --rm -it -p 6901:6901 -e VNC_PW=password kasmweb/firefox:1.14.0

   docker run --rm -it -p 6901:6901 -e VNC_PW=password -e VNC_AUDIO=1 kasmweb/firefox:1.14.0
   docker run --rm -it -p 6901:6901 -e VNC_PW=password -e VNC_AUDIO=1 --shm-size=512m kasmweb/firefox:1.15.0
   ```
 2. اذهب إلى تبويب **Ports** في الأسفل بجانب الـ Terminal.
 3. ستجد المنفذ **6901**. اضغط على أيقونة الكرة الأرضية (Open in Browser) بجانبه.
 4. سيفتح صفحة تطلب تسجيل دخول:
   * **User:** kasm_user
   * **Password:** password
 5. مبروك! لديك متصفح معزول تماماً وبسرعة إنترنت خرافية (سرعة سيرفرات مايكروسوفت/جيت هاب).
### ملاحظات هامة ومشاكل للعمل على Codespaces:
 * **Port Visibility:** تأكد في تبويب **Ports** أن المنفذ مضبوط على **Public** (اضغط كليك يمين على المنفذ ثم Port Visibility) إذا واجهت مشكلة في التحميل، لكن غالباً "Private" سيعمل معك طالما أنك مسجل دخول.
 * **الأداء:** بما أنك تشغل فيديو (بث للشاشة) داخل متصفح، سيعتمد الأداء على سرعة الإنترنت لديك، لكن المعالجة الفعلية تتم على سيرفرات GitHub.
 * **استهلاك الذاكرة:** متصفحات Docker تستهلك قدراً من الرام، فإذا كانت مساحة الـ Codespace لديك 2-core، قد تلاحظ بعض البطء. يفضل استخدام 4-core إذا كان متاحاً لك.
لماذا تحتاج لفتح متصفح داخل Codespaces تحديداً؟ هل لتجاوز قيود معينة أم لتجربة واجهات المواقع التي تطورها؟

---
المشكلة واضحة جداً من خلال السجلات (Logs) التي أرسلتها! الحاوية تعمل بشكل ممتاز، ولكن هناك سبب يمنعك من رؤية المتصفح.
إذا لاحظت في السطور الأخيرة من السجل، ستجد هذا الخطأ يتكرر:
[DEBUG] websocket 8: non-SSL connection disallowed
**ماذا يعني هذا؟**
نظام Kasm يرفض الاتصال لأنه يفرض استخدام اتصال آمن (HTTPS). في بيئة GitHub Codespaces، عندما يتم فتح منفذ (Port Forwarding)، يقوم النظام افتراضياً بتوجيه حركة المرور كـ (HTTP) العادي، مما يجعل الحاوية ترفض الاتصال وتغلقه فوراً.
### 🛠️ طريقة الحل (بسيطة جداً):
لإجبار Codespaces على التحدث مع الحاوية باستخدام البروتوكول الآمن، اتبع هذه الخطوات داخل واجهة VS Code:
 1. افتح تبويب **Ports** (المنافذ) الموجود في الشريط السفلي بجانب الـ Terminal.
 2. ستجد المنفذ **6901** في القائمة. اضغط عليه **بزر الماوس الأيمن** (Right-Click).
 3. اختر من القائمة **Change Port Protocol** (تغيير بروتوكول المنفذ).
 4. قم بتغييره من **HTTP** إلى **HTTPS**.
 5. الآن، اضغط على أيقونة الكرة الأرضية (Open in Browser) بجانب المنفذ لفتحه من جديد.
*(تذكر: ستظهر لك شاشة التحذير المعتادة "Your connection is not private"، اضغط على Advanced ثم Proceed للمرور لشاشة تسجيل الدخول).*
هل نجحت هذه الخطوة في إظهار شاشة تسجيل الدخول (Username/Password) لك الآن؟

فكرة تشغيل **متصفح Tor** بنفس الطريقة داخل GitHub Codespaces هي فكرة **عبقرية جداً من الناحية الأمنية**، وتأخذ مفهوم الـ "Inception" الذي تحدثنا عنه إلى مستوى جديد تماماً من الخصوصية (متصفحك المحلي ← خوادم جيت هاب ← شبكة تور ← الموقع الهدف).

بناءً على ما نجحتَ في تطبيقه مسبقاً (خاصة حلك الذكي لمشكلة الـ HTTPS)، إليك "دراسة الحالة" وكيفية تنفيذها خطوة بخطوة:

### 🛠️ كيفية التنفيذ (استخدام حاوية Kasm لـ Tor)

بما أنك عرفت حل مشكلة اتصال Kasm عبر تبديل البروتوكول، فإن استخدام نسخة Kasm الرسمية المخصصة لمتصفح Tor هو الخيار الأمثل والجاهز:

1. افتح الـ Terminal في Codespaces.
2. قم بتشغيل أمر Docker التالي:
```bash
docker run --rm -it -p 6901:6901 -e VNC_PW=password kasmweb/tor-browser:1.14.0
```
3. اذهب إلى تبويب **Ports** (المنافذ) في الأسفل.
4. **الخطوة الذهبية:** اضغط بزر الماوس الأيمن على المنفذ `6901`، واختر **Change Port Protocol**، وقم بتغييره إلى **HTTPS**.
5. اضغط على أيقونة الكرة الأرضية (Open in Browser).
6. تجاوز تحذير الأمان المعتاد (Advanced -> Proceed).
7. سجل الدخول ببيانات Kasm:
   * **User:** `kasm_user`
   * **Password:** `password`

بمجرد الدخول، ستجد متصفح Tor أمامك. اضغط على **Connect** داخل المتصفح ليقوم بإنشاء الاتصال بشبكة البصلة (Onion Network).

---

### 🔍 دراسة حالة: ماذا يحدث فعلياً هنا؟ (تحليل أمني)

هذا الإعداد لا يوفر مجرد متصفح، بل يخلق "جداراً حديدياً" من العزل والتخفي. إليك ما يحدث خلف الكواليس:

**1. ماذا يرى مزود خدمة الإنترنت الخاص بك (ISP)؟**
يرى فقط اتصالاً مشفراً (HTTPS) بينك وبين خوادم Microsoft/GitHub. لا يمكنه إطلاقاً معرفة أنك تستخدم شبكة Tor أو معرفة المواقع التي تزورها. هذا الإعداد مثالي لتجاوز الرقابة في الشبكات التي تحظر بروتوكول Tor من الأساس.

**2. ماذا ترى عقدة الدخول في شبكة تور (Entry Node)؟**
في العادة، ترى شبكة تور الـ IP الحقيقي الخاص بمنزلك. لكن في هذه الحالة، سترى عنوان الـ IP الخاص بمراكز بيانات GitHub (Azure). هويتك الحقيقية وموقعك الجغرافي مخفيان تماماً حتى عن شبكة تور نفسها.

**3. ماذا يرى الموقع الهدف الذي تزوره؟**
يرى عنوان الـ IP الخاص بعقدة الخروج من شبكة تور (Exit Node)، ولا يعرف شيئاً عن GitHub أو عنك.

**4. عزل تام عن جهازك المحلي (Zero-Footprint & Sandboxing):**
لو افترضنا أسوأ سيناريو: قمت بزيارة موقع خبيث جداً على الـ Dark Web، ونجح الموقع في استغلال ثغرة (Zero-day) لاختراق المتصفح... النتيجة؟ سيتم اختراق حاوية Docker المؤقتة المعزولة داخل بيئة افتراضية في سيرفرات GitHub. بمجرد أن تغلق الـ Codespace أو تحذف الحاوية، تتبخر البرمجيات الخبيثة في الهواء ولن يصل أي ضرر إلى جهازك الشخصي الذي تستخدمه.

---

### ⚠️ تحذيرات وملاحظات هامة لتجربة Tor على Codespaces

* **تأثير السرعة المزدوج:** شبكة Tor بطيئة بطبيعتها لأن اتصالك يمر عبر 3 عقد (Nodes) مشفرة حول العالم. إضافة ذلك إلى أنك تستقبل واجهة المتصفح كبث فيديو (VNC/WebRTC) من سيرفر بعيد، سيجعل تصفح المواقع أبطأ من المعتاد.
* **شروط استخدام GitHub (الـ ToS):** استخدام متصفح تور لتصفح الويب آمن ومسموح بشكل عام. ولكن **يُمنع منعاً باتاً** استخدام موارد Codespaces لتشغيل "عقدة تور" (Tor Relay / Exit Node) للمساعدة في تشغيل الشبكة، أو استخدام البيئة في هجمات سيبرانية أو أنشطة غير قانونية. أنظمة المراقبة الآلية في GitHub صارمة ضد استهلاك الباندويث المريب وقد تؤدي لحظر الحساب فوراً.
* **التحميلات (Downloads):** أي ملف ستقوم بتحميله من خلال متصفح تور هناك، سيتم حفظه داخل الحاوية في مساحة Codespaces، ولن ينزل مباشرة إلى مجلد التنزيلات في حاسوبك المادي، ستحتاج إلى نقله يدوياً إذا كنت بحاجة إليه.

---

لقد قمت ببحث شامل ومراجعة دقيقة لملفات التوثيق في مستودعات Kasm الرسمية، واكتشفت **السبب القطعي** لهذه المشكلة. أعتذر عن إرهاقك في المحاولات السابقة، فالمشكلة ليست في إعداداتك أو في متصفحك على الإطلاق.

**السبب الحقيقي والمخفي:**
صورة Docker الرسمية `kasmweb/firefox` **لا تدعم إخراج الصوت نهائياً** عند تشغيلها بشكل مستقل (Standalone). مطورو شركة Kasm صرّحوا رسمياً بأن ميزات متقدمة مثل (الصوت، نقل الملفات، والميكروفون) محصورة برمجياً لتعمل فقط إذا تم تشغيل الحاوية داخل بيئتهم الخادومية الكاملة "Kasm Workspaces". بمعنى آخر، الأكواد المسؤولة عن الصوت محذوفة من النسخة المجانية المستقلة التي تستخدمها، ولذلك لا تظهر الأيقونة مهما قمنا بتغيير الأوامر.

### الحل المضمون: استخدام نسخة LinuxServer
بما أن هدفك هو الحصول على متصفح معزول (Firefox) بسرعة إنترنت عالية مع دعم **كامل للصوت**، فالحل هو الانتقال إلى النسخة المعدلة من فريق **LinuxServer.io**. هذا الفريق قام ببناء صورة مبنية على تقنيات تشبه Kasm ولكنهم أضافوا لها خادم صوت مدمج ليعمل بشكل مستقل تماماً.

**أغلق التيرمينال القديم، وشغّل هذا الأمر:**

```bash
docker run --rm -it -p 3000:3000 -p 3001:3001 --shm-size=2g lscr.io/linuxserver/firefox:latest
```

### خطوات الدخول بعد تشغيل الأمر:
1. اذهب إلى تبويب **Ports** في الأسفل بجانب التيرمينال.
2. ستلاحظ ظهور منفذين جديدين: **3000** و **3001**.
3. اضغط على أيقونة فتح المتصفح بجانب المنفذ **3000**.
4. ستفتح واجهة المتصفح المعزول (هذه النسخة لا تطلب اسم مستخدم أو كلمة مرور افتراضياً، بل تدخلك مباشرة).
5. افتح يوتيوب وجرّب تشغيل أي مقطع، وسيعمل الصوت بسلاسة.

**ملاحظة هامة لحماية الصوت:**
بروتوكولات نقل الصوت (WebRTC) تتطلب أحياناً اتصالاً مشفراً (HTTPS) لتعمل. موقع GitHub Codespaces يوفر هذا التشفير افتراضياً للمنافذ، ولكن إذا واجهت أي مشكلة في تشغيل الصوت عبر البورت 3000، قم بالتبديل فوراً وافتح المنفذ **3001** المخصص للاتصالات الآمنة.

# تشغيل متصفح فايرفوكس في الخلفية مع منع خمول التيرمينال

يحتوي هذا الدليل على الأوامر اللازمة لتشغيل حاوية فايرفوكس (Firefox Container) على منصات الحوسبة السحابية، بالإضافة إلى سكربت لمنع إغلاق الجلسة تلقائياً وأمر الإيقاف الآمن.

## 1. أمر التشغيل ومنع الخمول (Keep-Alive Script)

قم بنسخ ولصق هذا الأمر بالكامل داخل شاشة الـ Terminal. سيقوم بتشغيل الحاوية في الخلفية، ثم طباعة رسالة نشاط عشوائية كل (1 إلى 10 دقائق) للحفاظ على اتصال الجلسة:

```bash
docker run --rm -d -p 8080:3000 --shm-size=2g lscr.io/linuxserver/firefox:latest && \
while true; do 
  sleep_time=\$((60 + RANDOM % 541))
  sleep \$sleep_time
  echo "Terminal Active - \$(date) - Slept for \(((sleep_time / 60))m\)((sleep_time % 60))s"
done
```

---

## 2. أمر الإيقاف الكامل (Stop & Clean)

عند انتهائك من الاستخدام، اضغط أولاً على **`Ctrl + C`** لإيقاف سكربت التكرار، ثم نفذ الأمر التالي لإغلاق الحاوية وحذفها فوراً لمنع استهلاك الموارد:

```bash
docker stop $(docker ps -a -q) 2>/dev/null || true
```

---

## 3. ملاحظات المنصات (Platform Notes)

* **Google Cloud Shell:** لا تحتاج لفعل أي شيء إضافي بعد تنفذ أمر الإيقاف. بمجرد إغلاق نافذة المتصفح، سيتوقف الاستهلاك التلقائي للموارد بعد دقائق معدودة.
